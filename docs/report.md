# Итоговый отчёт — OzonHack, Трек 2

## Резюме

Предложена **модульная система кросс-белт сортировки** из 4 параллельных модулей (400 направлений), обеспечивающая целевой поток **100 000 товаров/час** при площади **~9 700 м²** (49% от лимита 20 000 м²). Решение включает полный цикл работы с КТЯ, антизатор, отказоустойчивость и дискретно-событийную симуляцию на SimPy.

---

## 1. Постановка задачи (интерпретация команды)

Сортировочный центр должен распределять поток товаров по 400 направлениям с размещением в КТЯ. Ключевая сложность — не механизм сброса, а **согласованность процессов**: сортировка, заполнение тары, смена КТЯ, устойчивость к перегрузкам.

## 2. Выбранный подход

**Кросс-белт сортировщик** в конфигурации 4 × 100 направлений:

- Промышленно проверенная технология для 100–500+ ячеек
- Параллельные модули дают отказоустойчивость и масштабируемость
- Автоматическая смена КТЯ снижает зависимость от ручного труда

Альтернативы (шаттл, поперечные отводы, роботы) отклонены из-за площади, стоимости или ограничений по габаритам при 400 направлениях.

## 3. Принцип работы

1. Товар → входной распределитель → модуль по балансировке загрузки
2. Сканер штрихкода → WMS → ячейка 1..400
3. Индукция на контур кросс-белта
4. Транспорт и сброс в ячейку
5. Накопление в КТЯ → уплотнение → автосмена

Подробнее: [concept.md](concept.md)

## 4. Ключевые узлы

| Узел | Функция |
|------|---------|
| Входной распределитель | Буферизация, балансировка 4 модулей |
| Кросс-белт модуль | Транспорт и сброс в 100 ячеек |
| Сканер + WMS-интерфейс | Идентификация и маршрут |
| Ячейка сброса + КТЯ | Накопление, уплотнение, смена |
| Система управления | PLC + сервер, антизатор |

## 6. Симуляция и проверка

Реализована модель на **SimPy** с 5 сценариями (600 с модельного времени):

| Сценарий | Пропускная способность | Корректность | Перенаправления |
|----------|------------------------|--------------|-----------------|
| baseline | 99 792 т/ч (99,8%) | 99,45% | 0,54% |
| overload_125pct | 125 832 т/ч | 99,05% | 0,93% |
| chute_failure | 99 738 т/ч | 98,48% | 1,50% |
| module_degraded | 100 614 т/ч | 99,53% | 0,47% |
| hotspot_jam | 100 110 т/ч | 96,52% | 3,46% |

Запуск: `python simulation/run.py --scenario all --duration 600`

## 7. Характеристики по критериям ТЗ

| Характеристика | Как учтено |
|----------------|------------|
| Производительность | 4 модуля, расчёт + симуляция |
| Масштабируемость | Модульная архитектура +N×100 направлений |
| Устойчивость к заторам | Антизатор, буферы, сценарий hotspot |
| КТЯ | Полный цикл в модели и конструкции |

| Сложность | 4 стандартных модуля vs 1 монолит на 400 |

## 8. Ограничения
