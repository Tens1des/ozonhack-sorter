# Концепция решения: модульный кросс-белт сортировщик

## 1. Выбор конструкции

Выбран **кросс-белт сортировщик (Cross-Belt Sorter)** в модульной конфигурации:

- 4 независимых модуля по 100 направлений
- Каждый модуль — замкнутый овальный контур с тележками и поперечными лентами
- Товар сбрасывается в боковую ячейку без остановки контура

### Почему кросс-белт

| Критерий | Обоснование |
|----------|-------------|
| 400 направлений | Промышленный стандарт для 100–500+ ячеек |
| Разброс габаритов | Мягкая обработка, подходит для 10 г – 5 кг |
| Производительность | Масштабируется параллельными модулями |
| Площадь | Компактнее линейных отводов при большом числе направлений |
| Отказоустойчивость | Модуль можно изолировать без остановки всей системы |

## 2. Принцип работы

```
Входной поток → Распределитель → [Модуль 1..4]
                                      ↓
                              Сканер штрихкода
                                      ↓
                              Запрос в WMS
                                      ↓
                              Индукция на контур
                                      ↓
                              Транспорт по контуру
                                      ↓
                              Сброс в ячейку (1..400)
                                      ↓
                              КТЯ → контроль → смена
```

### Этапы обработки одного товара

1. **Подача** — товар поступает на распределитель входного потока
2. **Идентификация** — сканирование штрихкода (50 мс)
3. **Маршрутизация** — запрос в WMS, получение ячейки 1..400 (15 мс)
4. **Индукция** — размещение на тележку модуля (120 мс на линию)
5. **Транспорт** — движение по контуру до точки сброса (~11 с в среднем)
6. **Сброс** — поперечная лента выталкивает товар в ячейку (180 мс)
7. **КТЯ** — накопление, уплотнение, смена при заполнении

## 3. Ключевые узлы

### 3.1 Входной распределитель

- Буферизация пиков входного потока
- Балансировка между 4 модулями по загрузке
- Пропускная способность: ≥ 30 т/с (запас к 27,8 т/с)

### 3.2 Кросс-белт модуль

| Параметр | Значение |
|----------|----------|
| Контур | Овал 58 × 26 м |
| Тележки | ~90 на модуль |
| Скорость контура | 2,5 м/с |
| Линии индукции | 4 |
| Ячейки сброса | 100 |

### 3.3 Система идентификации

- Лазерные сканеры на каждой линии индукции
- Дублирование: повторное сканирование при ошибке
- Таймаут WMS: 100 мс → повтор / отбраковка

### 3.4 Ячейки сброса и КТЯ

- Каждая ячейка: накопительный лоток + КТЯ на роликовом столе
- Датчики: уровень заполнения, наличие КТЯ, переполнение
- **Уплотнение**: пневматический толкатель (коэффициент 0,92 объёма)
- **Смена КТЯ**: автоматическая (14 с) — подъём полного, установка пустого

### 3.5 Система управления

- PLC на модуль + центральный сервер маршрутизации
- Антизатор: при перегрузке ячейки — перенаправление в соседнюю
- При отказе ячейки — overflow в резервную зону модуля

## 4. Обязательные процессы (по ТЗ)

| Процесс | Реализация |
|---------|------------|
| Сортировка | 4 кросс-белт модуля, 400 ячеек |
| Контроль заполнения КТЯ | Датчики уровня + ПО, порог 88% → уплотнение |
| Уплотнение | Пневмо-толкатель, цикл 0,4 с |
| Снятие заполненных КТЯ | Автоматический подъём, 14 с |
| Установка пустых КТЯ | Магазин пустой тары, синхронно со снятием |

## 5. Нештатные режимы

| Ситуация | Реакция системы |
|----------|-----------------|
| Переполнение ячейки | Антизатор → соседняя ячейка модуля |
| Отказ ячейки | Исключение из маршрута, overflow |
| Деградация модуля | Снижение скорости, перераспределение потока |
| Отказ WMS | Локальный кэш маршрутов (последние 10 000 SKU) |
| Затор на индукции | Буфер входа, сигнал upstream замедлить подачу |

## 6. Безопасность

- Физические ограждения вокруг движущихся частей
- Световые барьеры на зонах обслуживания КТЯ
- Аварийные кнопки E-Stop на каждом модуле
- Режим обслуживания с блокировкой приводов (LOTO)

## 7. Масштабируемость

- **По направлениям**: добавление модулей по 100 ячеек
- **По потоку**: добавление линий индукции или модулей
- **По площади**: модульная компоновка, не требует перепроектирования
